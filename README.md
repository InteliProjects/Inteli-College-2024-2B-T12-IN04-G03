## 📜 Descrição

&nbsp;&nbsp;&nbsp;&nbsp;O problema identificado é a ausência de um sistema automatizado para o monitoramento contínuo dos equipamentos no laboratório de processos metalúrgicos, o que resulta em altos custos operacionais, desgaste prematuro dos equipamentos e interrupções frequentes no funcionamento devido à supervisão inadequada. A solução proposta envolve o desenvolvimento de um dispositivo de baixo custo capaz de monitorar automaticamente as condições operacionais dos equipamentos, mesmo quando não estão em uso. O sistema monitorará variáveis críticas como temperatura, vibração e tempo de uso, permitindo a detecção precoce de falhas e a necessidade de manutenção preventiva, evitando problemas maiores e paradas inesperadas.
A solução será implementada como um sistema integrado aos equipamentos, funcionando continuamente e coletando dados em tempo real. Esses dados serão transmitidos para uma central de controle, gerando automaticamente alertas e relatórios de manutenção. O sistema terá uma interface amigável para facilitar o acompanhamento pelos técnicos e engenheiros.

## 📁 Estrutura de pastas

Dentre os arquivos e pastas presentes na raiz do projeto, definem-se:

- <b>assets</b>: aqui estão os arquivos relacionados a parte gráfica do projeto, ou seja, as imagens e vídeos que os representam (O logo do grupo pode ser adicionado nesta pasta).

- <b>document</b>: aqui estão todos os documentos do projeto, incluindo o manual de instruções (se aplicável). Há também uma pasta denominada <b>outros</b> onde estão presentes outros documentos complementares.

- <b>src</b>: Todo o código fonte criado para o desenvolvimento do projeto, incluindo firmware, notebooks, backend e frontend, se aplicáveis.

- <b>README.md</b>: arquivo que serve como guia e explicação geral sobre o projeto (o mesmo que você está lendo agora).
